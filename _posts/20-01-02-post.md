## 재귀 알고리즘
### 재귀
***  
- 재귀(Recursion)는 자신을 서브루틴으로 호출하는 함수로 문제를 해결 접근 방식이다.  
- 재귀 호출은 하위 단계의 문제(Sub problem)이 더 이상의 재귀 없이 해결되는 지점까지 계속된다.   
- Base case(재귀하지 않고 종료되는 시나리오)와 recurrence relation(베이스 케이스에 대한 다른 케이스)로 구성되어 무한 루프를 방지한다.  
- 재귀 호출 지점은 함수 내에 다양한 곳 위치한다.  

***
 [ReverseString](https://leetcode.com/problems/reverse-string/)

```cpp
class Solution {
public:
    void reverseString(vector<char>& s) {
        reverseStringRecursive(s,0,s.size()-1);
    }
    
    void reverseStringRecursive(vector<char>& s,int left,int right)
    {
        if(left>=right)return;//base case 왼쪽포인터가 오른쪽포인터를 넘어서면
        char temp = s[left];
        s[left] = s[right];
        s[right] = temp;
        reverseStringRecursive(s,++left,--right); //이 부분에서 증감연산자를 뒤에썼더니 오류가 났다. 함수가 작동하고 증감하는듯 하다. 추후 포스팅에 이부분을 알아보자.
    }
};
```

### 재귀함수  
***
 재귀함수 작성 순서  
1. 주어진 문제를 작은 문제들로 분할시킨다.  
2. 분할한 작은 문제들을 재귀적으로 해결할 함수 호출을 작성 
3. 재귀 함수 호출 작성시 처리부분 작성  
 즉,  문제를 비슷한 작은 문제로 분할하여 접근하는 분할정복이 필요하다.  

***
[Swap Nodes in Pairs](https://leetcode.com/problems/swap-nodes-in-pairs/)
```cpp
/**
 * Definition for singly-linked list.
 * struct ListNode {
 *     int val;
 *     ListNode *next;
 *     ListNode(int x) : val(x), next(NULL) {}
 * };
 */
class Solution {
public:
    ListNode* swapPairs(ListNode* head) {
        return swapPairsRecursive(head);
    }
    ListNode* swapPairsRecursive(ListNode* head){
        if(head == NULL) return NULL; // basecase :  마지막일시 null 반환 종료 
        if(head ->next ==NULL) return head; //basecase 2: 하나 남았을시 그대로 반환
        //swap 
        ListNode * temp = head ->next->next;//세번째 노드 임시저장   1->2->3
        head->next->next = head; //두번째 노드가 첫번째노드를 가르키게함.  1->2->1
        head = head->next; // 첫번째 노드를 두번째 노드로 변경한다.    2->1->2->1->...
        head->next->next = swapPairsRecursive(temp); //변경된 두번째노드의 next를 재귀함수의 리턴값으로 변경한다.  2->1->리턴값
        return head;//헤드값을 반환한다.
    }
};
```
